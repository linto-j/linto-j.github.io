100G Leaf-Spine Data Center Fabric

This project involved designing and deploying a high-performance 100G leaf-spine fabric using L3 underlay architecture with BGP routing to support scalable, resilient, and high-throughput workloads.

Architecture Overview

  • Spine-Leaf topology with 100G uplinks
  • L3 underlay using BGP for deterministic routing
  • Structured IP addressing strategy
  • Service leaf for north-south traffic control
  • Multi-tenant VLAN segmentation model

Why L3 Fabric Instead of L2?

A Layer 3 fabric was selected to eliminate spanning-tree dependency, reduce broadcast domain complexity, and enable deterministic routing behavior. This approach improves scalability and simplifies operational troubleshooting.

Routing Model

The underlay was built using BGP to provide dynamic route exchange between leaf and spine layers. This ensured scalable horizontal expansion while maintaining predictable routing paths.

Design Decisions

  • Chose L3 fabric over L2 to eliminate STP and improve scalability
  • Implemented BGP underlay for dynamic and deterministic routing
  • Designed segmentation for multi-tenant GPU workloads
  • Ensured high availability with no single point of failure

Key Outcomes

  • High-throughput east-west traffic optimization
  • Simplified troubleshooting model
  • Scalable architecture for future expansion
  • Production-grade resilience and stability
